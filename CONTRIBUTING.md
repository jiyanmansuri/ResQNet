# Contributing to ResQNet

Thank you for your interest in contributing to ResQNet! We welcome contributions of all forms—bug reports, feature ideas, documentation improvements, and code updates.

## Getting Started

1. Fork the repository and clone it locally.
2. Create a new branch for your feature or bug fix:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. Set up the local environment and verify all components run correctly (see [START.md](file:///c:/Users/jiyan/OneDrive/Desktop/ResQnet/START.md)).

## Coding Standard

*   **Python:** Follow PEP 8 guidelines for code style. Ensure docstrings are provided for new modules and public functions.
*   **React:** Component styles should use cohesive modular styles or central variables where applicable. Ensure elements are properly clean of unneeded re-render hooks.

## Submitting Pull Requests

1. Commit your changes locally.
2. Ensure commit messages are descriptive.
3. Push to your branch and open a Pull Request.
4. Ensure your PR description summarizes the exact changes made and how they were tested.
